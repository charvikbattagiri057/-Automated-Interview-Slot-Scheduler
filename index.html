Project Overview

What is AISS?

The Automated Interview Slot Scheduler (AISS) is designed to assign candidates to interview slots by validating constraints and evaluating utility scores. The system models interviewers and candidates, applies CSP backtracking to find valid assignments, and ranks results using a multi-criteria utility function — producing structured, explainable, and academically meaningful output.

🤖
AI Agent Design
A SchedulerAgent perceives the environment using the PEAS framework — tracking candidates, interviewers, and time slots — then formulates the problem as a state-space for downstream solving.
🔍
BFS Exploration
Breadth-first search explores the space of reachable (interviewer, slot) pairs per round type before the CSP runs, measuring node expansions and validating reachability.
🧩
CSP Backtracking
Variables are (candidate, round) pairs. Backtracking with MRV heuristic and preference-first domain ordering resolves conflicts and finds a valid, constraint-satisfying schedule.
📊
Utility Scoring
Each assignment is scored out of 10 across four criteria: preference alignment, expertise match, early slot bonus, and interviewer load balance — selecting the best feasible outcome.
Project Objectives

What This Project Achieves

01
Model the interview scheduling problem using AI-oriented state-space representation with candidates, interviewers, rounds, and time slots as core components.
02
Apply BFS search to explore the reachable space of (interviewer, slot) combinations per round type before assigning, measuring node expansions and runtime.
03
Use CSP backtracking with MRV heuristic to enforce hard constraints — no double-booking, expertise matching, and slot conflict detection — across all assignments.
04
Score each final assignment using a multi-criteria utility function and generate a ranked schedule optimising preference alignment and load balance.
05
Generate structured outputs — terminal reasoning trace, schedule.json, schedule.csv, and an HTML showcase — suitable for academic review and demonstration.
06
Build a clean, modular Python pipeline integrating all four CFAI course outcomes into one cohesive, explainable intelligent system.

Course Outcome Mapping

Module / CO Area Academic Requirement AISS Implementation
CO1Problem Formulation & Agent Model Represent the problem as a state space with actions, transitions, goals, and costs using Python data structures and a PEAS agent. SchedulerAgent perceives interviewers, candidates, and slots; formulates variables as (candidate, round) pairs with an explicit goal state and cost function.
CO2Search & Algorithmic Reasoning Implement and analyse graph/state-space search algorithms with empirical profiling of node expansions and runtime. CO4Utility & Decision Making Design decision-making agents that select the best outcome using utility functions and evaluate performance under bounded computation. UtilityScorer assigns scores (0–10) per assignment across four weighted criteria: preference alignment, expertise match, early slot bonus, and load balance.
CO5Reasoning Under Uncertainty Consider dynamic or uncertain scenarios in decision logic and reasoning pipelines. The modular architecture supports extension toward probabilistic availability and uncertain interviewer schedules; the agent model is designed to handle incomplete information.
CO6Integrated AI Pipeline Build integrated AI reasoning pipelines combining representation + search + CSP + decision logic with explainable outputs. The main pipeline integrates all four modules sequentially — agent perception → BFS exploration → CSP backtracking → utility scoring — with a full reasoning trace exported as JSON, CSV, and HTML.
Core Features

What the System Does

👥
Multi-Interviewer Modelling
Each interviewer has a defined expertise type (HR / Technical / Managerial) and a set of available time slots, modelled as Python dataclasses with assignment tracking.
🗓️
Candidate Preference Slots
Candidates declare their required interview rounds and preferred time slots. The CSP domain ordering prioritises preferred slots, maximising alignment in the final schedule.
🔄
Multi-Round Scheduling
Supports HR, Technical, and Managerial rounds simultaneously. Each (candidate, round) pair is an independent CSP variable assigned without conflicts across all rounds.
⚡
Conflict Detection
Hard constraints prevent any interviewer from being double-booked and any candidate from being scheduled for two interviews in the same slot, across all round types.
📝
Reasoning Trace
Every assignment attempt, successful assignment, and backtrack is logged with variable name, domain size, chosen value, and reason — forming a complete explainability trail.
📤
Structured Exports
Final schedule exported as schedule.json and schedule.csv alongside the terminal trace and HTML dashboard, covering all required academic output formats.
System Workflow

How It Works — Step by Step

🧠
Agent Perceives Environment
SchedulerAgent reads all interviewers, candidates, and rounds; builds the slot universe and formulates the problem state space.
🔍
BFS Explores Reachability
Breadth-first search scans all reachable (interviewer, slot) pairs per round type using a FIFO queue, counting node expansions.
🧩
CSP Assigns Slots
Backtracking solver selects variables by MRV, orders domains by preference, checks all constraints, and assigns or backtracks.
📊
Utility Scores Results
Each assignment is evaluated across four criteria and assigned a score out of 10, producing the final ranked schedule.
📤
Outputs Generated
Schedule printed to terminal with full reasoning trace, then exported as JSON and CSV files for analysis and academic review.
